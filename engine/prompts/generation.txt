## CRITICAL: Task-Driven Simulation Design

This simulation is TASK-DRIVEN, not open-ended. The candidate must:
1. Complete specific tasks within 45 minutes
2. Extract information from agents to complete each task
3. Navigate relationships to unlock information

**Each task should require information from 1-3 agents.** The candidate cannot complete tasks without talking to the right people and asking the right questions.

## Your Task
Generate a complete simulation environment:

1. **Company Details**: Use EXACTLY the company name "{org_name}". Base the company description on the context provided above.

2. **Scenario Tension**: A realistic workplace challenge that creates urgency and context for the tasks.

3. **Task Queue (3-5 tasks)**: Each task must have:
   - Clear deliverable (decision, document, recommendation)
   - `required_info`: List of agents who hold pieces needed to complete it
   - `completion_type`: How candidate proves completion:
     - "chat_validation": Must show/explain work to validating_agent
     - "artifact_edit": Must update a section of the artifact
     - "deliverable": Must describe/present solution in chat
   - `validating_agent_id`: Who signs off on completion
   - `deliverable_prompt`: What the agent will ask for to validate

4. **Agent Cast (4-5 agents)**: Each agent must have:
   - Unique personality and relationship to candidate
   - `task_knowledge`: List of tasks they have info for, with:
     - What info they hold
     - Condition for sharing ("asked directly", "relationship > 0.6", "only after they share X")
   - `hidden_information`: Info they won't share easily

5. **Background Chatter (2-3 conversations)**: Pre-existing conversations between agents that they may reference:
   - Agent A and B discussed topic X
   - Creates realism ("I was just talking to Sarah about this...")
   - Can reveal info or create complications

6. **Initial Inbox**: 3-4 email threads that relate to tasks

7. **Stress Injects**: 3-4 timed interruptions at minutes 5, 15, 25, 35

8. **Artifact**: A role-appropriate work document with editable sections linked to tasks:
   - PM: PRD with sections (Problem, Solution, Requirements, Timeline, Risks)
   - Analyst: Analysis report (Executive Summary, Data Findings, Recommendations)
   - Eng Manager: Technical spec (Overview, Architecture, Implementation, Timeline)
   - Sales: Proposal (Client Needs, Solution, Pricing, Next Steps)
   - Ops Lead: Process document (Current State, Issues, Proposed Changes, Rollout)

   Each section should have:
   - `section_id`: Unique identifier
   - `title`: Section heading
   - `content`: Pre-filled content (some correct, some needing fixes based on info from agents)
   - `linked_task_id`: Which task requires editing this section (optional)
   - `editable`: true/false

   IMPORTANT: Some sections should contain INCORRECT or INCOMPLETE information that the candidate needs to fix by gathering info from agents. This creates the core gameplay loop.
